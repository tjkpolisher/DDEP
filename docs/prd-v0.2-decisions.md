# PRD v0.2 Decisions

## 확정된 검토 포인트

- MVP 1차 사용자는 팀 내부자입니다.
- 진단 도메인은 6개로 충분합니다.
- 수학/물리 기초 도메인은 타겟 계층에 비해 과합니다.
- 검색 에이전트 구현도 MVP에 포함합니다.
- 문항 DB 구축은 별도 Phase로 분리합니다.
- Python 패키지 버전 관리는 `uv`를 사용합니다. 개발 환경에 `uv`가 설치되지 않았을 경우 우선 `uv`를 사용자 개발 환경(OS) 등에 설치하는 작업을 먼저 수행하세요.

## 6개 진단 도메인

1. 기체/공력
2. 전장/하드웨어
3. 제어
4. 소프트웨어
5. 자율비행/AI
6. 제작/운용

## Phase 분리

1. Foundation: 프로젝트 기본 동작, 기술 스택, 저장소 규칙
2. Question DB: 문항 스키마, 태그 체계, 초기 문항 세트
3. Diagnosis Engine: 점수 업데이트, 적응형 출제, 취약 개념 추출
4. Result Report: 도메인별 프로파일, 로드맵, 재진단 UX
5. Search Agent: 검색, 필터링, 추천 자료 스코어링
6. Service MVP: 인증, 저장, 배포, 내부 베타 운영
