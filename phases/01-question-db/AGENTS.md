# AGENTS.md - Phase 01 Question DB

## 목적

드론 개발 역량 진단에 사용할 문항 DB의 스키마, 태그 체계, 초기 문항 세트를 구축합니다.

## 범위

- 6개 도메인 기준 문항 스키마 정의
- `domain`, `subdomain`, `difficulty`, `prerequisites`, `concept_tags`, `answer_type` 필드 정의
- 객관식/단답형 중심의 초기 문항 작성
- 문항 검수 상태와 출처 메타데이터 설계
- 문항 seed/import 방식 구현

## 도메인 규칙

- 수학/물리 기초는 독립 도메인으로 만들지 않습니다.
- 필요한 기초 개념은 `prerequisites` 또는 `concept_tags`에 포함합니다.
- 문항은 CAELUM 내부자가 실제 드론 개발 역량을 점검하는 데 필요한 수준으로 작성합니다.
- 너무 학술적인 퍼즐형 문제보다 개발 의사결정과 트러블슈팅에 연결되는 문항을 우선합니다.

## 완료 조건

- 6개 도메인 모두 최소 문항 세트를 가집니다.
- 각 문항은 난이도와 선행 개념을 포함합니다.
- 문항 DB는 코드 상수가 아니라 seed 데이터 또는 DB 레코드로 관리됩니다.
- 문항 검수 상태를 구분할 수 있습니다.

## 금지 사항

- MVP 핵심 채점 경로를 LLM 주관식 채점에 의존하지 않습니다.
- 출처와 검수 상태 없는 문항을 production seed로 취급하지 않습니다.
- 한 도메인에만 문항이 과도하게 몰리지 않게 합니다.

## 산출물

- 문항 스키마
- 초기 seed 데이터
- 문항 import/validate 명령
- 문항 작성 가이드
