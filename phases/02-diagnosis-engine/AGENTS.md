# AGENTS.md - Phase 02 Diagnosis Engine

## 목적

문항 응답을 바탕으로 도메인별 점수, concept mastery, 취약 개념, 다음 출제 방향을 계산합니다.

## 범위

- IRT-lite 또는 Bayesian score 기반 점수 업데이트
- 도메인별 0~100 점수 산출
- concept tag별 mastery 추적
- 오답 시 prerequisite 방향 후퇴
- 연속 정답 시 난이도 상승 또는 인접 심화 개념 이동
- 진단 세션 상태 모델링

## 알고리즘 기준

- MVP에서는 과도한 psychometrics보다 설명 가능하고 디버깅 쉬운 알고리즘을 우선합니다.
- 결과는 총점보다 도메인별 프로파일과 학습 순서 도출에 유리해야 합니다.
- 같은 입력에 대해 재현 가능한 결과를 내야 합니다.

## 완료 조건

- 샘플 문항 응답으로 6개 도메인 점수가 계산됩니다.
- 취약 concept tag와 prerequisite chain이 도출됩니다.
- 적응형 출제 후보를 선택할 수 있습니다.
- 알고리즘 단위 테스트가 있습니다.

## 금지 사항

- 점수 계산을 LLM 응답에 직접 의존하지 않습니다.
- 알고리즘 파라미터를 설명 없이 흩어진 매직 넘버로 두지 않습니다.
- 결과 리포트 UI 구현을 이 Phase의 핵심 작업으로 끌고 오지 않습니다.

## 산출물

- 진단 세션 모델
- 점수 업데이트 함수
- 취약 개념 추출 함수
- 적응형 출제 후보 선택 함수
- 단위 테스트
