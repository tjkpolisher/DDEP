# AGENTS.md - Phase 03 Result Report

## 목적

진단 결과를 CAELUM 내부 사용자가 바로 이해하고 다음 학습 행동으로 옮길 수 있는 리포트로 제공합니다.

## 범위

- 6개 도메인 점수 프로파일
- 강점/약점 요약
- 취약 concept tag 표시
- 선행 개념 기반 학습 순서 생성
- 재진단 대상 도메인/개념 제안
- 이전 진단과 비교할 수 있는 데이터 구조

## 리포트 기준

- 단일 총점 중심 화면을 만들지 않습니다.
- "자율비행 점수가 낮다"에서 멈추지 않고 "PID -> IMU -> sensor fusion -> EKF -> attitude control"처럼 다음 순서를 보여줍니다.
- 내부 MVP이므로 과한 마케팅 문구보다 진단 근거와 액션이 명확해야 합니다.

## 완료 조건

- 사용자는 자기 약점과 다음 학습 순서를 한 화면에서 파악할 수 있습니다.
- 결과 데이터는 검색 에이전트 Phase의 입력으로 재사용됩니다.
- 리포트 구성 요소가 테스트 가능한 데이터 모델을 기반으로 렌더링됩니다.

## 금지 사항

- 결과를 통과/실패 또는 하나의 등급으로 단순화하지 않습니다.
- 검색 에이전트가 검증하지 않은 외부 자료를 리포트에 직접 넣지 않습니다.
- UI 텍스트에 알고리즘의 불확실성을 숨기지 않습니다.

## 산출물

- 결과 리포트 스키마
- 리포트 UI 또는 API 응답
- 학습 로드맵 생성 로직
- 재진단 추천 로직
